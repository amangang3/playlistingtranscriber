Cartelera Teatral Madrileña · Automated Transcription

Turning 270-year-old theater ledgers into a spreadsheet

This tool reads scanned pages from an 18th-century Spanish theater record book and extracts every performance into neat rows, so historians can sort, count, and compare without typing a thing.

The problem

Between roughly 1748 and 1800, Madrid's two main public theaters — Teatro de la Cruz and Teatro del Príncipe — kept meticulous records of every play, every night: the title, the author, the company director, the day of the week, and how much money came in at the door. Those records were later compiled into a printed reference book called the Cartelera Teatral Madrileña.

Today those pages live in PDFs. Each page is a dense, two-column table of plays and dates, printed in tiny type, with section headers, footnotes, and running lists of ticket receipts. A human researcher would need weeks to type one volume into a spreadsheet by hand. This tool does it in an afternoon.

What you put in: a scanned PDF of the Cartelera.
What you get out: an Excel file with one row per performance — 4,000 to 5,000 rows per PDF — ready to filter, sort, or feed into further analysis.

How each step works

1

Turn each page into a high-resolution picture

The computer opens the PDF and takes a detailed snapshot of every page — roughly the same resolution as a good magazine scan. This is important: the print is tiny, and a blurry picture would mean the AI can't read the words.

Split two-page spreads down the middle

Each scanned page is actually two pages from the original book sitting side by side. The tool slices the image vertically so each half can be read as its own single page. This doubles the number of pictures the AI has to process, but the AI can focus on one page at a time and make fewer mistakes.

4

Ask the AI to read the page and fill in a form

This is the heart of the tool. The image is sent to a modern vision AI model (Google's Gemini) along with detailed instructions: "You're looking at a page from an 18th-century Spanish theater record. For each performance date you see, fill in this form: theater name, date, play title, author, ticket receipts, director."

The AI reads the page, understands the layout (dates on the left, titles in the middle, receipts on the right), and returns a structured answer — not a paragraph of text, but a filled-out form for every single performance it found. A page with one play that ran ten nights becomes ten rows, each with its own date and receipt total.

5

Remember context between pages, then save to Excel

Sections in the book span many pages. A director's name might appear once at the top of a section and never again for the next twenty pages. The tool remembers this context: once it sees "María Hidalgo, company director", it fills that in for every following performance until a new director shows up.

When every page is done, the tool assembles the results into an Excel spreadsheet — one row per performance, tidy columns, ready to open.

Why this is hard

The print is dense

Each page packs 80–100 performances into two tight columns of 18th-century typography. One wrong guess and a whole row slides.

The layout isn't consistent

Section headers appear only on some pages. A play that ran ten nights gets one line and ten date numbers, which must be expanded into ten rows.

Spanish, accents, historical spelling

Play titles use archaic Spanish spelling and accented characters. The tool is instructed to preserve them faithfully.

Quality checking

An included evaluation script compares the AI's output against existing hand-typed records for a few test pages, so you can see how accurate the extraction really is before trusting it.
